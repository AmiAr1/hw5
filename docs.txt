1. то такое структуры данных?

По сути, это способы хранить и организовывать данные,
чтобы эффективней решать различные задачи. Данные можно представить по-разному.
В зависимости от того, что это за данные и что вы собираетесь с ними делать,
одно представление подойдёт лучше других.


2. Зачем нужны структуры данных?
Данные являются важной сущностью, а структуры позволяют хранить,
пользоваться, оптимизировать их в организованной форме.

3.Hash Map — основан на хэш-таблицах,
 реализует интерфейс Map (что подразумевает хранение данных в виде пар ключ/значение).


4. Linked List — базовая динамическая структура данных в информатике,
состоящая из узлов, каждый из которых содержит как собственно данные,
так и одну или две ссылки на следующий и/или предыдущий узел списка.

5.1 Linked List - использовать когда происходит активная работа (вставка/удаление)

5.2
Hash Map - использует хеш-таблицу для хранения карточки,
обеспечивая быстрое выполнения, реализует интерфейс Map (хранение данных в виде пар ключ/значение).
